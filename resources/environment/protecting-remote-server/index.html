Protecting Remote Server

Tips to protect remote server after fresh install, first thing you want to do is to update all packages and reboot system to apply kernel patches.

~> apt update && apt upgrade -y

Then disable logging in with cleartext passwords

~> vi /etc/ssh/sshd_config
# PermitRootLogin (yes) => (without-password|no)
# PasswordAuthentication no 
~> systemctl restart sshd

Install Fail2Ban and create SSHD jail

~> apt install fail2ban
~> cat /etc/fail2ban/jail.d/defaults-debian.conf                                                                                                                                                                     [11:22:10]
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 600
~> systemctl restart fail2ban
~> fail2ban-client status sshd                                                                                                                                                                                       [11:23:10]
Status for the jail: sshd
|- Filter
|  |- Currently failed: 3
|  |- Total failed:     508
|  `- File list:        /var/log/auth.log
`- Actions
   |- Currently banned: 15
   |- Total banned:     15
   ...

BONUS: NGINX Fail for spamming 403/404 attempts (Nikto?)

~> cat /etc/fail2ban/filter.d/nginx-403.conf
[Definition]
failregex = ^<HOST> -.*"(GET|POST|HEAD).*HTTP.*" 403
ignoreregex =
~> cat /etc/fail2ban/filter.d/nginx-404.conf
[Definition]
failregex = ^<HOST> -.*"(GET|POST|HEAD).*HTTP.*" 404
ignoreregex =
~> cat /etc/fail2ban/jail.d/nginx.conf
[nginx-403]

enabled = true
port    = http,https
filter  = nginx-403
logpath = /var/log/nginx/access.log
maxretry = 5
findtime = 300
bantime = 1800

[nginx-404]

enabled = true
port    = http,https
filter  = nginx-404
logpath = /var/log/nginx/access.log
maxretry = 10
findtime = 300
bantime = 1800

That’s it, remember to update your system frequently, make backups and reboot it from time to time.
